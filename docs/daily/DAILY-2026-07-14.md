# 工作日报（2026-07-14）

## 一、问题背景

| 项 | 内容 |
|----|------|
| 模块 | 前置过滤 export / 货架末行投影误报 |
| 问题 | 站立时手腕投影落入每个货架最底一行（n×n 的 layer==rows），误告警频发 |
| 影响 | triple90+torso160 方案 FP 仍偏高，且集中在末行 token |
| 环境 | 分支 `try/speed-filter`，28-clip baseline-local |

## 二、根因分析

| 类型 | 原因 |
|----|------|
| 几何 | 透视下站立人体手腕 Y 坐标偏低，与末行货位多边形重叠 |
| 数据 | 双货架机位合并标注后需 per-shelf 识别底行，非全局单一末行 |
| 算法 | 原碰撞对所有 box 一视同仁，站立时未排除末行 |

## 追加：末行识别修正（max layer per shelf）

### 问题

`2-5-1/clip_0009` 双货架中 shelf 91 仅标 9 格（layer 1–3），旧逻辑 `layer==grid_rows(4)` 导致末行为空，`Box_4043/4044/4045`（layer=3）站立时仍误报。

### 修复

`event_engine/shelf_grid.py`：末行 = **该 shelf 已标注层的 max(layer)**。shelf 91→4043–4045；shelf 92→3077–3080。

### 重跑结果（同 export 目录）

| 指标 | 修正前末行逻辑 | 修正后 |
|------|----------------|--------|
| FP | 289 | **218** |
| FN | 15 | **18** |
| 召回 | 90.38% | **88.46%** |

`clip_0009` 帧 5635–5673 的 `Box_4044` 连续误报已消除。

对比 MD：`docs/compare-ankle80-triple90-torso160-excl-bottomrow-vs-torso160.md`

## 追加：肩髋膝角替代 torso160 站立特征

### 问题背景

| 项 | 内容 |
|----|------|
| 模块 | 速度前置过滤站立门控 |
| 问题 | torso160 用 ∠(肩,髋,踝)，用户希望用肩髋膝角更直接描述躯干-大腿折叠 |
| 目标 | 在 ankle@80+triple90 下复现 torso160 指标（FN=10, FP≈311） |

### 根因 / 设计

| 类型 | 原因 |
|----|------|
| 算法 | 原 `skeleton_angles` 无 ∠(肩,髋,膝) 聚合字段 |
| 配置 | 需单独网格扫描阈值，不能沿用 torso160 的 160° |

### 实现修改

修改文件：

- `event_engine/skeleton_angles.py`：新增 `SHOULDER_HIP_KNEE_ANGLE_DEFS`、`_knee_center`、`shoulder_hip_knee_angle_{mean,min,max}`
- `scripts/data/validate_prefilter_shoulder_hip_knee_stance28.py`：27 组阈值网格扫描
- `docs/prefilter-ankle80-triple90-shknee-stance-experiment.md`：实验记录

推荐规则：`ankle_max@80 + triple90 + shoulder_hip_knee_angle_min>=140`

### 验证结果

| 指标 | torso160 | shknee140 | Δ |
|------|----------|-----------|---|
| TP | 146 | 146 | 0 |
| FN | 10 | 10 | 0 |
| FP | 311 | 310 | -1 |
| 召回 | 93.59% | 93.59% | — |

导出包：`localdata/export/rule-speed-prefilter-ankle-max80-triple90-shknee140-prod-test`
