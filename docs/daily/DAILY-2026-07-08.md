## 二、代码改动（工作区，待提交）

### 1. 事件复核导航与 pause 竞态（`08-event-review.js`、`11-playback-controls.js`）

- **`filteredPlaybackEvents`（漏报）**：时间线无命中时，按 `externalPlaybackAccuracyOverlay` 未检出段起点生成可导航占位事件。
- **`navigateReviewEvent`**：筛选模式用 `getActiveFilteredIndex()` 定位；边界直接 `return`；中间项 `seekToEvent(list[nextIdx])`。
- **`seekToEvent`**：先 `pause` 再设 `activeEventKey`；`seek` 完成后再次锁定 `playbackEventLinkExact`。
- **`video pause` 监听**：`playbackEventLinkExact` 为真时跳过 `syncActiveEventFromPlaybackPosition`，避免覆盖显式跳转选中项。
- **筛选下拉 `change`**：切换后 `blur()`，减少键盘 ⬆⬇ 与原生下拉冲突。

### 2. 帧对齐与紫色高亮修复（`09-playback-events.js`、`10-render-collision.js`）

- 新增 **`frameEntryByIdx`**、**`pinnedEventFrameIdx`**、**`playbackOverlayFrameIdx`**。
- **`seekToTimestamp`**：传入 `frame_idx` 时优先 `frameByTime` 命中并 **`renderFrameEntry` 强制渲染目标帧**，不再只靠 `currentTime` 二分找帧。
- **`getReviewBoxHighlightContext` / `drawAnnotationBoxes`**：事件钉住时，标真紫填与漏报黑边对齐**事件帧**。
- **`redrawCurrentFrame`**：钉住事件时按事件 `frame_idx` 重绘，避免 `seekToEvent` 末尾再次落回上一帧。

### 3. 上传评估 overlay 漏报/误报判定（`10-render-collision.js`）

- **`isPlaybackEventInMissSegment`**：外部 overlay 下漏报段不要求已标真；段起点帧或 token 匹配即纳入筛选。
- **`isPlaybackEventFalseAlarm`**：与 overlay `allAlarms` 及标真段覆盖关系对齐，减少误报筛选漏项。

### 4. 缓存版本（`index.html`）

- `08-event-review.js` v71、`09-playback-events.js` v73、`10-render-collision.js` v77、`11-playback-controls.js` v75。
