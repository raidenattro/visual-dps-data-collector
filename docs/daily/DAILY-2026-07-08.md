## 一、问题背景

| 项 | 说明 |
|----|------|
| 典型记录 | `rtmpose-m/1-1-1/clip_0013_start_00-11-22_rtmpose_m` |
| 标真落盘 | `localdata/review/1-1-1/clip_0013_start_00-11-22.089_end_00-21-28__00-11-22__full/event_review.json` |
| 推测 baseline | `localdata/export/rule-baseline-prod-test/clip_0013_start_00-11-22_rtmpose_m.json` |
| 现象 A | 准确率页筛选「漏报/误报」跳转回放后，事件复核 ⬆⬇ 无法切换相邻同类型条目 |
| 现象 B | 970 帧标真仅 `Box_1006`，画面却出现 **2 个紫色货框**（含相邻 `Box_1007` 黄边） |
| 现象 C | 切换漏报/误报筛选卡顿；漏报段导航条目与评估段数不一致 |
| 控制台验证 | `eventFrame: 970`，`renderedFrame: 969`，`confirmed: [Box_1006]` |

**根因归纳**：

| 类别 | 原因 |
|------|------|
| 事件导航 | 筛选队列索引错误；漏报段无时间线事件时队列为空；`pause` 回调覆盖 `seekToEvent` 的 `activeEventKey` |
| 紫色双框 | 预览视频 seek 吸附关键帧差 1 帧；标真合并段与选中事件范本两路叠加 |
| 评估 overlay | 碰撞与告警未分列表；漏报/误报筛选未与 overlay 段、告警列表完全对齐 |
| 性能 | `getAccuracyGroundTruthSegments` 每次筛选全量比对；`invalidatePlaybackAccuracyOverlay` 触发全量重绘 |

### 关键改动摘要

**导航与帧对齐**（`08/09/10/11`）

- 权威 `frame_idx` + `seekToTimestamp` 强制 `renderFrameEntry`，避免 480p 预览视频关键帧漂移。
- 逐帧 ⬅▶ 控件；`getPinnedPlaybackEvent` 支持误报/漏报占位事件钉住。
- `buildFalseAlarmQueueEvents` / `buildMissSegmentQueueEvents`：每段/每条告警一条，与评估计数一致。

**评估 overlay**（`api/eval_diagnostics.py` 等 + `10-render-collision.js`）

- overlay 输出 `collisions` 列表；回放用推测碰撞黄框 + 告警红框。
- `getAccuracyGroundTruthSegments` 合并 review 标真段与 overlay `detected` 状态，并加缓存。

**复核 UI**（`index.html` + `style.css`）

- 「标记复核完成」置顶；批量操作折叠；事件 dock 可滚动。
