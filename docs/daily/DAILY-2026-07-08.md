# 工作日报（2026-07-08）

## 一、问题背景

| 项 | 内容 |
|----|------|
| 模块 | 手臂延长探针（hand_extended）规则导出 / 准确率评估 |
| 问题 | 需在 28 条优质样本上重跑 α=0.1/0.15/0.2，输出与 `rule-baseline-prod-test` 同格式的逐帧 JSON，供上传推测准确率计算 |
| 影响 | 无导出 JSON 时无法在准确率页/CLI 对延长探针做 is_picking 对标真评估 |
| 环境 | try/elbow-extend 分支；现场参数 `pose_frame_interval=2`，`alarm_min=3`，`alarm_cooldown=0`；机位 8 个；标签 单人+无遮挡 · 已复核 · 有标真 |

## 二、根因分析

| 类型 | 原因 |
|----|------|
| 架构 | 既有 `export_rule_baseline_frames.py` 仅走 `InferCollisionProcessor`（手腕），不支持 `hand_extended` 与 `extension_ratio` |
| 算法 | `simulate_frame_events_infer_collision` 未暴露探针参数，无法在同一 infer-collision 帧枚举路径下切换延长探针 |
| 数据 | 准确率评估需要 clip 级 JSON：`is_picking`（告警）、`rule_collisions`、`rule_alarm_collisions`，格式须与 baseline 一致 |

## 三、实现修改

### 手臂延长探针规则 JSON 导出

修改文件:

- `event_engine/collision_sim.py`
- `scripts/data/export_rule_baseline_frames.py`

修改内容:

- `simulate_frame_events_infer_collision` 增加 `probe_mode` / `extension_ratio`；`hand_extended` 时使用 `CollisionProcessor`，默认 `wrist` 仍用 `InferCollisionProcessor`
- 导出脚本新增 CLI：`--probe-mode hand_extended`、`--extension-ratio`
- manifest 记录探针类型与 α

修改原因:

- 在同一 skeleton frame_idx 帧枚举逻辑下导出延长探针结果，保证 JSON 字段与 baseline 对齐

影响:

- 可批量生成多 α 目录供准确率工具直接 `--dir` 评估

### 执行命令

```
python scripts/data/export_rule_baseline_frames.py --dry-run \
  --probe-mode hand_extended --extension-ratio 0.1 \
  --pose-frame-interval 2 --alarm-min 3 --cooldown 0

python scripts/data/export_rule_baseline_frames.py \
  --probe-mode hand_extended --extension-ratio 0.1 \
  --pose-frame-interval 2 --alarm-min 3 --cooldown 0 \
  --infer-size-record-root D:\work\workspace\git-repo\ShelfPickSense\data\data28-merged\Train \
  --out-dir localdata/export/rule-hand-ext-alpha010-prod-test

# 同理 0.15 → rule-hand-ext-alpha015-prod-test，0.2 → rule-hand-ext-alpha020-prod-test
```

### 结果

- Passed：三组均 28/28 条导出成功

- 告警帧合计（picking_frame_count 求和）：baseline 1910；α=0.1 → 1945；α=0.15 → 1924；α=0.2 → 1947
- JSON 字段抽检：`is_picking` = `bool(rule_alarm_collisions)`，`picking_prob`/`predicted_box_tokens` 为空
