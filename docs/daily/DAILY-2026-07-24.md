# 工作日报（2026-07-24）

## 一、问题背景

| 项 | 内容 |
|----|------|
| 模块 | 回放 Tab — 事件复核（标真 / event_review.json） |
| 现象 | ① 多人同框时标真无法记录「哪条骨架」；② 连续帧逐条按 Y 标真效率低；③ 复核侧栏文字与按钮挤压重叠 |
| 样例记录 | `1-1-1-_2/00000000907002400_seg07_11-23_to_12-47__0__full` |
| 环境 | Web SPA + FastAPI；`verified_true` 存于 `localdata/review/{机位}/{clip_key}/event_review.json` |

## 二、实现内容

### 2.1 person_id 标真落盘

| 层级 | 说明 |
|------|------|
| 后端 | `normalize_review_entry` 支持可选 `person_id`；toggle 标真时合并该字段；`enrich_events_with_review` 回读至事件列表 |
| 前端 | 骨架旁显示 `P{id}`；侧栏单选 / 点击骨架选择；标真写入 `verified_true` 条目 |
| 数据 | 例：`{ "event_type": "alarm", "frame_idx": 54, "box_tokens": ["Box_2013"], "confirmed_box_tokens": ["Box_2013"], "person_id": 1 }` |

**说明**：`person_id` 为帧内序号；跨帧关联标真时依赖 `person_track_id` 映射（见区间标真）。

### 2.2 区间增强标真

| 步骤 | 操作 |
|------|------|
| 1 | 当前帧 → **首帧**；在首帧选 **person_id** + **货框** |
| 2 | 当前帧 → **尾帧** |
| 3 | **区间标真**：对 `[首帧, 尾帧]` 内、货框与模板有交集的全部 alarm/collision 事件批量标真 |

- 各帧写入相同 `confirmed_box_tokens`
- 各帧 `person_id` 按 `person_track_id` 自动解析；多人帧无法匹配时阻断并提示
- 首帧选取缓存为 `rangeAnnotTemplateSnapshot`，切换事件后不丢失暂选

### 2.3 复核区 UI

- 保留 **货框 token 状态行**（检测 / 已保存 / 暂选），移除冗余货框操作区（点选仍在画面）
- 滚动区分三块：**摘要** / **逐帧+标真操作** / **区间标真+批量**
- 增大间距、货框 token 独立浅底框、空 hint 不占位，缓解文字与边框挤压

## 四、验证步骤

| 项 | 操作 |
|----|------|
| 单人帧标真 | 选事件 → 选货框 → Y 标真 → 检查 `event_review.json` 含 `person_id` |
| 多人帧标真 | 首帧选 P0/P1 → Y 标真 → JSON 各条目 `person_id` 正确 |
| 区间标真 | 首帧 54、尾帧 59，选 Box_2013 + P0 → 区间内多条一次性标真 |
| UI | 侧栏无 token 与按钮重叠；区间面板与 person 提示不挤在一起 |
