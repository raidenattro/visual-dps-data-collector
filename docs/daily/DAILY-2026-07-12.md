## 补充：回放特征加载拖慢视频（2026-07-12）

### 一、问题背景

| 项 | 内容 |
|----|------|
| 模块 | 回放打开记录、视频首屏加载 |
| 问题 | 新增骨骼特征后，打开记录加载视频明显变慢 |
| 影响 | 视频/骨架与特征 API 争抢后端，首屏卡顿 |
| 环境 | Web SPA + FastAPI 同步端点 |

### 二、根因分析

| 类型 | 原因 |
|----|------|
| 后端 | `/playback-features` 每请求读 Parquet 多帧并重算速度/角度，默认还读手腕 Parquet |
| 前端 | `renderFrameEntry` 每帧调用 `updatePlaybackSkeletonFeaturesUi`；特征加载完成又 `redrawCurrentFrame()` 形成请求+重绘风暴 |
| 架构 | 打开记录时 `redrawCurrentFrame` 与视频加载并行，首屏即触发重型 API |

### 三、实现修改

修改文件：

- `api/playback_features_service.py`：进程内 LRU 缓存 `(record_id, frame_idx, include_wrist)`；默认 `include_wrist=false` 跳过手腕 Parquet
- `api/routes/http.py`：新增查询参数 `include_wrist`
- `web/app/13-playback-skeleton-features.js`：视频就绪前不请求；播放中仅用缓存；暂停 debounce 280ms；移除 `redrawCurrentFrame` 循环
- `web/app/10-render-collision.js`：播放中跳过特征 UI 更新；新增 `repaintFeatureTrackLabelsOnly` 轻量刷新标签
- `web/app/06-records.js`：视频加载完成后延迟 700ms 再 `enablePlaybackSkeletonFeatureFetch`

### 四、改动文件索引

| 文件路径 | 修改说明 |
|----|----|
| api/playback_features_service.py | LRU 缓存 + 可选手腕 |
| api/routes/http.py | include_wrist 参数 |
| web/app/13-playback-skeleton-features.js | 延迟/防抖/播放守卫 |
| web/app/10-render-collision.js | 播放跳过 API + 轻量标签重绘 |
| web/app/06-records.js | 视频就绪后启用特征拉取 |

### 五、验证步骤

| 项 | 结果 |
|----|------|
| `python -m py_compile api/playback_features_service.py` | Passed |
| 浏览器打开记录测首屏 | Not tested |

### 六、性能影响

| 指标 | 修改前 | 修改后 |
|----|----|----|
| 打开记录时特征 API | 立即每帧请求 | 视频就绪+700ms 后，且仅暂停时 |

| 后端同帧重复计算 | 每次请求重算 | LRU 命中直接返回 |
