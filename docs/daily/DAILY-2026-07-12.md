# 工作日报（2026-07-12）

## 一、问题背景

| 项 | 内容 |
|----|------|
| 模块 | 前置速度门控 ankle_max@80 + triple90 |
| 问题 | 相对 baseline 仍多 2 段漏报；蹲取时脚速大被门控，但上肢角度不满足 triple90 |
| 影响 | 召回 92.95% vs baseline 94.23% |
| 环境 | 28-clip prod-test，pose_frame_interval=2 |

## 二、根因分析

| 类型 | 原因 |
|----|------|
| 门控 | 超速且未满足 triple90 → block，蹲取贴地时 wrist_elevation 极低 |
| 特征 | `skeleton_angles.py` 原无膝角；2D 俯视膝角 P50 标真/误报均 ~174°，整体区分弱 |
| 数据 | 蹲取段膝角 min 更低（如 159–169 段 knee_angle_min P50≈140°） |

## 三、实现修改

### 站立/非站立姿态门控豁免实验

修改/新增文件：

- `event_engine/skeleton_angles.py`：膝角 + leg_span_ratio + hip_knee_ankle_vertical_ratio
- `scripts/data/screen_prefilter_stance_proxy28.py`：阶段 0 特征筛查
- `scripts/data/validate_prefilter_stance_exempt28.py`：28-clip 姿态豁免网格
- `scripts/data/export_prefilter_triple_angle_upload.py`：`--stance-feature` / `--stance-threshold`
- `docs/prefilter-stance-proxy-screen.md`、`docs/prefilter-stance-exempt-speed-gate.md`
- `docs/prefilter-foot-ankle-speed-gate.md` §11

门控逻辑：仅 `knee_angle_mean >= 120`（站立）时速度+triple90 门控可 block；蹲姿豁免。

## 补充：正式采用肩髋踝方案（2026-07-12 晚）

### 一、问题背景

| 项 | 内容 |
|----|------|
| 模块 | 速度预过滤门控（ankle_max@80 + triple90） |
| 问题 | 蹲取时踝速高导致漏报；膝角在俯视 2D 下区分度弱 |
| 决策 | 用户要求采用 **∠(肩, 髋, 踝)** 作为站立判定特征 |
| 影响 | 导出默认、文档主推荐统一为 `torso_leg_angle_mean≥160` |

### 二、根因分析

| 类型 | 原因 |
|----|------|
| 算法 | 俯视 2D 下膝角/肩髋踝 P50 均接近伸直（~170°），单特征区分蹲站有限 |
| 数据 | 蹲取段踝速仍高，需姿态豁免而非单纯降速阈 |

### 三、实现修改

**导出默认**（`export_prefilter_triple_angle_upload.py`）：
- `DEFAULT_STANCE_FEATURE = torso_leg_angle_mean`
- `DEFAULT_STANCE_THRESHOLD = 160`
- 输出目录：`rule-speed-prefilter-ankle-max80-triple90-torso160-prod-test`

**文档**（`prefilter-foot-ankle-speed-gate.md` §11）：
- 主推荐从 `knee_angle_mean≥120` 改为 `torso_leg_angle_mean≥160`

### 四、改动文件索引

| 文件路径 | 修改说明 |
|----|----|
| scripts/data/export_prefilter_triple_angle_upload.py | 默认肩髋踝 stance 参数与示例 |
| docs/prefilter-foot-ankle-speed-gate.md | §11 主推荐改为 torso160 |
| localdata/export/rule-speed-prefilter-ankle-max80-triple90-torso160-prod-test/ | 新导出包 |
| localdata/export/compare_*torso160*.md | 对比报告 |
