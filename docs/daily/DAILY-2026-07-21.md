## 一、问题背景

| 项 | 内容 |
|----|------|
| 模块 | 回放 Tab — 骨架 overlay 与配套 MP4 对齐 |
| 现象 | **播放时**骨架与视频同步；**暂停、逐帧 ±1、事件跳转**时骨架超前或循环跳动；仅 **含 `video_start_pts_sec` 的新 manifest** 明显（旧数据无该字段时往往正常） |

| 环境 | Web SPA + FastAPI；timeline 已由 `normalize_frame_timestamps` 从 0 均匀分布 |

## 二、根因分析

### 2.1 数据与时间轴（ffprobe 验证）

脚本 `scripts/data/compare_frame_timing.py` 结论：

| 轴 | 含义 |
|----|------|
| `timestamp_sec` / timeline | 骨架索引时间，0 起算 |
| ffprobe PTS | ≈ timeline + **`video_start_pts_sec`**（恒定偏移） |
| 连续播放 rVFC `mediaTime` | 含容器 PTS，需 **减 PTS** 才回到 timeline |
| HTML5 `currentTime`（逐帧 seek 到 timeline） | 走解码/media 轴；`currentTime=10.0` 呈现帧约等于 ffprobe 234，而 timeline 10.0 对应 manifest 帧 ~250 |

因此：**播放**用 `mediaTime − PTS` 正确；**暂停 seek 到 `hit.t`** 时视频画面偏早 ~PTS×fps 帧，骨架若仍按 timeline 查帧则**超前约 17–19 帧**。

### 2.2 前端架构（迭代前）

| 路径 | 行为 | 问题 |
|------|------|------|
| 播放 | `playbackRenderLoop` → `resolveRvfcMediaTime` → `renderPlaybackFrameAtTime(lite)` | 正确 |
| 暂停/跳转 | `redrawCurrentFrame` → `renderFrameEntry(full)`，layout 重算 | 与播放不同轴、空间/时间双偏 |
| 自适应 PTS / authority 混用 | 多轮修补后互相打架 | 播放与跳转无法同时稳定 |

### 2.3 逐帧箭头「异常循环」

1. 逐帧 seek 完成后延迟 **`seeked`** 仍触发 `renderSkeletonSyncedToVideo({ setAuthority: true })`，用含 PTS 的 `currentTime` 覆盖 authority，帧号跳变 ~17 帧。  
2. 钉住事件时用 **timeline 时间** 与 **`videoSeekTimeForFrameIdx`（timeline+PTS）** 比较，误触发 `realignPlaybackToPinnedEvent()` 反复 seek。

## 三、最终实现方案

### 3.1 三条时间轴分工

| 场景 | 视频 seek | 骨架查帧 |
|------|-----------|----------|
| **连续播放 / 刚 pause** | — | `resolveRvfcMediaTime(lastPlaybackMediaTimeSec)`，始终减 PTS |
| **暂停逐帧 / 事件跳转** | `videoSeekTimeForFrameIdx` = timeline + PTS | seek 后 `waitPresentedVideoFrame` → `resolveRvfcMediaTime` → `renderPlaybackFrameAtTime(full)` |
| **进度条拖动** | timeline 轴（不加 PTS） | `seeked` + rVFC；清除 PTS seek 标志 |

### 3.2 播放期 layout 冻结

- pause 时 `cancelPlaybackRenderLoop({ preserveLayout: true })`，`pausedPlaybackLayout` 供 full 模式复用，避免 `getDisplayLayout()` 空间错位。

### 3.3 逐帧跳转防循环

- `seeked`：存在 `playbackAuthorityFrameIdx` 时直接 return，避免二次覆盖。  
- `syncActiveEventFromPlaybackPosition`：漂移检测用 **`videoEl.currentTime`** 与 **`videoSeekTimeForFrameIdx`** 比较；事件 sync 传 **timeline `displayT`**。  
- `playbackVideoClockUsesPtsSeek` + `timelineSecFromVideoClock()`：PTS seek 后正确还原 timeline；播放/拖条时清除标志。  
- `getCurrentPlaybackFrameIdx` 优先 authority，避免 ±1 步进基准帧错误。
