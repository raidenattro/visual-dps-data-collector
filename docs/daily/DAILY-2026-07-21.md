# 工作日报（2026-07-21）

## 一、问题背景

| 项 | 内容 |
|----|------|
| 模块 | 地面标定（spatial）与脚部 floor_xy 轨迹 |
| 目标 | 将 Handoff 项目脚部足迹能力整合进 visual-dps-datacollect |

## 二、实现概览

### 1. 核心模块 `spatial_pose/`

| 文件 | 说明 |
|------|------|
| `calibration.py` | 单应（homography）标定加载/保存、控制点、RMSE |
| `floor_projection.py` | 踝点选取、floor_xy 投射、EMA 平滑 |
| `grid.py` / `preview.py` / `schema.py` | 地面网格、预览、配置 schema |

### 2. 采集与存储

- `config.json` → `spatial.enabled`、`spatial.calibration_dir`
- `collect_core.py`：采集时按 `camera_slug` 写入 `foot_uv_px` / `floor_xy_m`
- `pose_store.py`：`timeline.parquet` 新增列 `foot_u/v_px`、`floor_x/y_m`、`raw_floor_x/y_m`（向后兼容 optional 列）
- `manifest.json` 增加 `spatial` 摘要节点

### 3. API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/spatial/cameras` | 可标定机位列表 |
| GET/PUT | `/api/spatial/calibration/{slug}` | 读写标定 |
| GET | `/api/spatial/calibration/{slug}/preview-frame` | 推理分辨率预览帧 |
| POST | `/api/spatial/calibration/{slug}/preview` | 预览网格与 RMSE |
| GET | `/api/records/{id}/spatial` | 记录 + 运行时标定 |

### 4. Web 地面标定页

- 新页签「地面标定」：`web/spatialCalibrate.js`
- 10 点交互标点、physical 参数、预览网格、保存至 `localdata/spatial/{slug}.json`
- `preview-frame` 按推理高（默认 480）缩放，与 RTMPose 采集分辨率一致

### 5. 回放 UI

- `web/app/14-playback-floor-map.js`：Ground Map 面板、地面网格 overlay
- **足部轨迹动画**（Handoff 风格）：
  - 视频 canvas：橙色渐变轨迹（仅画至当前帧）、踝点→判定点虚线、蓝色 FOOT 标记
  - 播放时随视频延伸，**暂停时轨迹与判定点同步停止**
  - Ground Map 同步渐进式轨迹
- 工具栏开关：「地面网格」「足部轨迹」

### 6. CLI / 离线补算

| 脚本 | 说明 |
|------|------|
| `scripts/spatial/calibrate_from_video.py` | OpenCV 交互标定 |
| `scripts/spatial/enrich_record_floor_xy.py` | 单条 v2 记录补算 timeline |
| `scripts/spatial/enrich_manifest_floor_xy.py` | 按 export manifest 批量补算 + 回写 export JSON |

文档：`docs/spatial-calibration.md`；模板：`examples/spatial.template.json`

### 7. 测试

- `tests/test_spatial_calibration.py`
- `tests/test_floor_projection.py`

## 三、问题修复

| 问题 | 原因 | 修复 |
|------|------|------|
| Web 标定「未返回视频帧」 | API 返回 dict 而非 base64；`preview-frame` 路由被 `{slug}` 吞掉 | 修正返回格式；路由提前注册 |
| 标定与采集分辨率不一致 | 原视频 2560×1440 标点用于 852×480 骨架 | preview-frame 按 infer_height 缩放；加载旧标定时缩放控制点 |
| 回放地面网格对不齐 | manifest 缺 infer 尺寸；播放路径未画网格 | 从 timeline 读 infer 尺寸；`drawSkeletonPlaybackOnly` 叠加网格 |

## 四、28-clip 批量补算（rule-baseline-local-prod-test）

```
python scripts/spatial/enrich_manifest_floor_xy.py
```

| 结果 | 数量 | 说明 |
|------|------|------|
| 成功 | 2 | 机位 `1-1-1`（已有标定，RMSE≈3.66 px） |
| 跳过 | 26 | 7 个机位尚无 `localdata/spatial/{slug}.json` |
| 错误 | 0 | 28 条源记录均可定位 |

补算写入：`timeline.parquet` + export clip JSON 的 `floor_x_m` 等字段。

## 五、timeline 存储与兼容性

- v2 记录包：`manifest.json` + `timeline.parquet` + `skeleton.parquet`
- floor 字段为 **增量 optional 列**；旧代码按列名 `.get()` 读取，无 floor 列时不报错
- 碰撞重算路径 `load_all_frames` → `write_timeline_parquet` 会保留已有 floor 字段
