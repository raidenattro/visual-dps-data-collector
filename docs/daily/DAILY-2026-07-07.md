## 一、问题背景

| 项 | 说明 |
|----|------|
| 典型记录 | `rtmpose-m/1-1-1/00000001139000000_clip_1m20s_5m00s_rtmpose_m` |
| 规模 | 5485 帧、3603 事件、Parquet schema v2 |
| 原片 | 2560×1440（约 78MB） |
| 现象 | 打开回放后骨架加载慢；播放时骨架跟不上人物、明显卡顿 |
| 关联问题 | 视频加载状态不清；无效预览 MP4（约 258B）无法播放；播放中货框/碰撞高亮异常 |

**根因归纳**：

| 类别 | 原因 |
|------|------|
| 视频 | 2560×1440 浏览器解码压力大；缺少 480p 预览转码与坏文件校验 |
| 骨架 | 播放中每帧走 `async renderAtTime`（含网络预取、全量货框+碰撞重绘） |
| 渲染 | 底部「播放」与 `video play` 事件各启一条 `requestAnimationFrame` 循环 |
| 索引 | Parquet 记录打开时拉取全量 timeline JSON，启动慢 |
| 碰撞 | 播放热路径若实时跑 `CollisionProcessor`，每帧开销极大 |

## 二、代码提交（已入库）

### 1. 回放预览视频转码与轻量 timeline API（`0535408`）

- **新增** `video_transcode.py`：
  - 高分辨率源视频按 `config.json` → `video.transcode_height`（默认 **480p**）异步转码为 H.264 预览；
  - 校验最小体积（50KB）+ OpenCV 可读帧，自动删除无效 preview（排除 258B 空壳 MP4）；
  - 转码失败时回退原片。
- **`api/record_service.py`**：
  - `playback_video_path_for_record`：仅返回有效预览，否则原片；
  - `playback_video_prepare_status`：查询/触发异步转码进度；
  - `persist_record_video`：采集落盘时过高分辨率自动转预览高度。
- **`api/routes/http.py`**：
  - `GET /api/records/{id}/video/preview/status`：转码进度轮询；
  - `GET /api/records/{id}/video?original=1`：强制原片；
  - `GET /api/records/{id}/timeline?light=1`：极轻时间轴（仅 `frame_idx` / `timestamp_sec` 等索引列）。
- **`pose_store.py`**：新增 `load_timeline_index`，Parquet 只读索引列。

### 2. 回放性能优化与预览转码等待 UI（`cc61a1f`）

**打开记录**

- `prefetchAllPlaybackChunksInBackground`：await 全量骨架分块预取，底部提示 `加载骨架 N%`；
- `buildSyntheticFrameIndexFromManifest`：`frame_interval=1` 时本地合成时间轴，零网络拉取；
- `prepareAndLoadRecordVideo`：转码轮询 + 舞台加载遮罩 + 预览失败自动回退原片；
- 加载完成提示 `骨架已就绪`；视频显示实际分辨率（期望 480×… 而非 2560×1440）。

**播放热路径**

- 唯一渲染循环 `ensurePlaybackRenderLoop`（`requestVideoFrameCallback` 优先，回退 `rAF`）；
- `syncRenderPlaybackFrame`：同步读 `frameCache` + `drawSkeletonPlaybackOnly`，不走 `async/await`；
- `frameIdxAtVideoTime`：O(1) 帧号换算；
- 播放时冻结布局 / canvas DPR=1；离屏 canvas **预烘焙静态货框**（避免每帧 stroke 30+ 框）；
- 播放中仅用**落盘碰撞**字段，跳过实时 `CollisionProcessor`；
- 播放中不做事件栏同步（暂停后全量重绘 + 事件定位）；
- `seeked` 在播放中短路，避免与渲染循环争抢主线程。

**UI**

- `#playback-stage-loading` 转圈遮罩 + 进度文案；
- 版本号 bump（`00-core` ~ `11-playback-controls` v71–75）。

## 四、验证步骤

1. **重启后端**（加载 `video_transcode.py`）。
2. 浏览器 **Ctrl+F5** 硬刷新。
3. 打开上述 5485 帧记录，确认：
   - 骨架：`加载骨架 100%` → `骨架已就绪`；
   - 视频：首次高分辨率应出现 `正在生成 480p 预览视频…`；
   - 底部：`已加载配套视频 480×…`（非 2560×1440）。
4. F12 → Network → `/video`：Content-Length 应 **远大于 258 字节**。
5. 点底部「播放」或空格，观察骨架是否跟上人物。

**若仍卡顿，区分两类现象**：

| 现象 | 可能原因 |
|------|----------|
| 骨架慢半拍 | 预览转码未生效，仍在解原片；检查 ffmpeg 是否安装 |
| 骨架冻结不动 | 骨架预取未完成或 `frameCache` 缺帧；看 Network `/frames` |
