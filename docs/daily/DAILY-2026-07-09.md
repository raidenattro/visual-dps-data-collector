# 工作日报（2026-07-09）

## 一、问题背景

| 项 | 内容 |
|----|----|
| 模块 | 姿态采集 / 碰撞误报过滤 |
| 问题 | 人走过货位时手腕短暂进框产生误报；现有手腕瞬时速度区分度弱（F1≈0.17），需从全骨骼运动特征（躯干/下肢速度）做二次过滤 |
| 影响 | 离线分析与后续段级过滤规则无法基于全身运动强度评估误报 |
| 环境 | Windows，Python，RTMPose COCO-17 骨架，v2 Parquet 记录包 |

## 二、根因分析

| 类型 | 原因 |
|----|----|
| 算法 | 手腕帧级速度受姿态抖动与手臂局部运动干扰，难以区分「停下拣货」与「走过」 |
| 架构 | 现有 `wrist_features.py` 仅覆盖左右手腕，缺少躯干/下肢聚合速度 |
| 数据 | 无 `skeleton_velocity.parquet` 与段级运动统计，无法做区分度分析 |
| 过滤 | 此前仅有位移/帧数 Combo 规则；本次采用纯速度段级上限过滤 |
