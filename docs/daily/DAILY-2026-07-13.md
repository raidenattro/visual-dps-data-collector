## 一、问题背景

| 项 | 内容 |
|----|------|
| 模块 | 回放 Tab、准确率页、标注存储、批量采集 |
| 问题 A | 预览 MP4 与骨架 overlay 存在固定时间偏移，拖动进度条后画面与骨骼不同步 |
| 问题 B | 手腕速度/置信度分散在独立侧栏，与速度门控特征对照不便 |
| 问题 C | 新货架标注写入 `localdata/json/annotations2`，前端/API 仍只读旧 `annotations` |
| 影响 | 回放复核、准确率诊断跳转、新批次标注采集无法闭环 |
| 环境 | Windows · main · localdata 本地数据 |

---

## 二、根因分析

| 类型 | 原因 |
|------|------|
| 回放 | 浏览器 `<video>` 时间轴从 0 起算，源片首帧 PTS 非零时 `timeSec × fps` 与骨架 `timestamp_sec` 错位 |
| 前端 | 未读取 manifest `video_start_pts_sec`，seek/render 未做 PTS 偏移校正 |
| UI | `12-playback-wrist-features.js` 与 `13-playback-skeleton-features.js` 功能重叠，手腕置信度未与 export 路径特征同源 |
| 数据 | `annotation_store` 仅映射 `json/annotations`，缺少 `annotations2` 作用域 |

---

## 三、实现修改

### 1. 回放视频与骨架 PTS 对齐（`b986626`）

修改文件：

- `video_transcode.py`：`probe_video_timing` 输出 `start_pts_sec` / `first_pts_sec`
- `collect_core.py`：采集 manifest 写入 `video_start_pts_sec`
- `api/record_service.py`：`enrich_manifest_playback_timing` 按需 ffprobe 补齐 PTS/时长
- `web/app/10-render-collision.js`：`playbackStartPtsSec()`，seek/render 减去 PTS 偏移
- `web/app/09-playback-events.js`、`01-layout-frames.js`：时间轴与事件定位同步
- `scripts/data/diagnose_playback_sync.py`、`compare_frame_timing.py`：对齐诊断脚本

影响：旧记录打开时 API 懒补齐 PTS；新采集自带 timing 字段。

### 2. 手腕置信度并入骨骼特征侧栏（`8a34f0a`）

修改文件：

- `api/playback_features_service.py`：`_build_wrist_confidence_map`，响应含 `wrist_confidence`
- `web/app/13-playback-skeleton-features.js`：手腕置信度表格、Stance/速度/角度统一卡片
- 删除 `web/app/12-playback-wrist-features.js`，精简 `index.html` 侧栏结构

影响：暂停时单面板展示置信度 + 门控预览 + 全骨骼特征，与准确率计算共用 score 阈。

### 3. annotations2 标注目录（`eff36b7`）

修改文件：

- `annotation_store.py`：`annotation2` / `annotations2` 作用域 → `json/annotations2`
- `api/record_service.py`、`api/routes/http.py`：列表/读写支持标注源切换
- `web/app/06-records.js`：回放标注 JSON 路径可选 `json/annotations2`
- `web/index.html`：标注源 UI

### 4. 新货架采集配置（`43d60eb`）

- 新增 `config.annotation2.json`：`annotation_dir=localdata/json/annotations2`
- `scripts/collect/batch_skeleton_collect.py`：支持 `--config config.annotation2.json`

### 5. 忽略 logs（`781734f`）

- `.gitignore` 增加 `logs/`
