# 工作日报（2026-07-20）

## 一、问题背景

| 项 | 内容 |
|----|------|
| 模块 | 速度门控（Speed Gate）/ 回放骨骼特征 / 误报对比脚本 |
| 问题 | ① `ankle_max_speed > 80` 为固定像素速度，换分辨率后不可复用；② 回放侧仅手腕特征，无法对照门控实验字段；③ 误报对比脚本与当前 export 目录结构不完全匹配 |
| 影响 | 多机位部署需重标像素阈值；现场调试门控需逐帧核对速度/角度；实验对比效率低 |
| 环境 | manifest28 prod-test，28 clip，`pose_frame_interval=2` |

## 二、根因分析

| 类型 | 原因 |
|----|------|
| 算法 | 门控阈值以绝对像素速度表达，未与 `hypot(infer_w, infer_h)` 归一化 |
| 数据 | manifest28 存在 852×480 与 853×480 两种 infer 尺寸，diag 差 <0.1% |
| 前端 | 原 `12-playback-wrist-features.js` 仅覆盖手腕特征，缺少踝速/角度/门控预览 |
| 工具 | `compare_export_false_alarms.py` 需适配多 export 包对比与 MD 输出 |
